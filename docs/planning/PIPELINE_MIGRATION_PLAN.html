5. Migration Steps

Three changes needed to close all gaps. Ordered by impact.

01
Migrate analyse-tiles.cjs
Copy Claude analysis generator into newleafsystem
Copy legacy source repo/pipeline/analyse-tiles.cjs to newleafsystem/pipeline/. Update Firebase config path. Add npm script: "analyse": "node pipeline/analyse-tiles.cjs". This fills the analyses collection that powers the 3 PositionDetail tabs.
02
Automate R2 → Firestore sync
Add sync-r2-to-firestore to daily cron
Add cron entry after OI enrichment: 40 14 * * 1-5 sync-r2-to-firestore.js (9:40am ET). This ensures Firestore tiles are always fresh after the daily scan completes. Currently manual — biggest automation gap.
03
Unified weekly command
Wire everything into npm run weekly
Update package.json scripts:
"weekly": "node pipeline/create-weekly-picks.js && node pipeline/analyse-tiles.cjs"
One command: create picks → generate Claude analysis → done. React app picks it up from Firestore.

9. Claude Analysis Schema

The analyses/{tileId} document structure generated by analyse-tiles.cjs. Powers the 3 tabs in PositionDetail.

Tab 1: Why This Trade?
strategyRationale:
• whyThisStrategy (2-3 sentences)
• whyTheseStrikes (strike selection logic)
• whyThisExpiry (time horizon reasoning)
• alternativesConsidered [{strategy, reason}]
Tab 2: Technical Analysis
technicalIndicators:
• rsi {value, signal, description}
• bollingerBands {upper, lower, width, signal}
• macd {macdLine, signalLine, histogram}
• movingAverages {sma20/50/100, crossover}
• impliedVolatility {currentIV, ivRank, ivPct}
• supportResistance [{level, strength}]
Tab 3: Theta & Risk
thetaDecaySchedule:
• dailyDecay [{daysToExpiry, dailyTheta, cumulative}]
• earlyCloseRecommendation

riskAnalysis:
• maxPainScenario
• earningsRisk, dividendRisk, eventRisk
• managementPlan (exit/adjustment rules)
