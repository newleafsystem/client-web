# NewLeaf System — Claude Code Guidelines

## Critical Rules

### NEVER edit `dist/`
The `dist/` directory is build output. Never create, edit, or copy files there directly.
- `vite build` generates `dist/` from source
- Static assets go in `public/` (Vite copies them to `dist/` automatically)
- The `copyStaticApps` plugin in `vite.config.js` copies `workbench/` → `dist/workbench/`
- `server.cjs` serves from `dist/` at localhost:3000

### NEVER hand-edit `workbench/nav-component.html`
This file is auto-generated by `scripts/build-workbench-nav.mjs`. Hand edits cause CSS drift.
- It SSR-renders `BrandBar.jsx` and embeds the full `src/trading/styles/nl-nav.css`
- To update nav styles: edit `nl-nav.css`, then run `node scripts/build-workbench-nav.mjs`
- To update nav structure: edit `BrandBar.jsx` + `navConfig.js`, then regenerate
- Auth handling comes from `scripts/workbench-nav-auth.js`
- Runtime interactivity from `scripts/workbench-nav-runtime.js`

## Architecture

### Two nav systems — one source of truth
- **React BrandBar** (`src/shared/components/BrandBar.jsx`) — used by SPA routes (/picks, /invest, /workbench/analysis)
- **Static nav-component.html** (`workbench/nav-component.html`) — used by workbench HTML pages (/workbench/*.html)
- Both must stay in sync. BrandBar is the source; the static version is generated from it.
- CSS source of truth: `src/trading/styles/nl-nav.css`

### Dev servers
- `node server.cjs` → localhost:3000 — serves from `dist/`, includes API proxy + R2 proxy
- `npm run dev` (vite) → localhost:5173 — serves React SPA with HMR
- Workbench static HTML pages are only available via server.cjs (port 3000)

### Build pipeline
1. `node scripts/build-workbench-nav.mjs` — regenerate static nav from BrandBar
2. `npm run build` (`vite build`) — build React SPA to `dist/`, copies `workbench/` to `dist/workbench/`

### Static files
- `public/` — favicons, robots.txt, sitemap.xml, privacy-policy.html, etc. Vite copies to `dist/`
- `workbench/` — standalone HTML apps (scanner, strategy pages). Copied to `dist/workbench/` by build plugin

## Code Patterns
- Firebase config: `src/firebase/config.js`
- Shared hooks: `src/shared/hooks/`
- Marketing pages: `src/marketing/`
- Trading/Invest pages: `src/trading/`
- Picks pages: `src/picks/`
