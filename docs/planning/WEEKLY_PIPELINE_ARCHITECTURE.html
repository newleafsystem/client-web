Weekly Pipeline Architecture

The definitive guide · Analysis-first design · April 2026

1. The Problem

The weekly pipeline produces 5 outputs. Today, Claude analysis runs last — after PDFs and pick JSONs are already generated from basic tile data. This creates inconsistency: the PDF says one thing, the app shows richer analysis.

Current problem: PDFs are generated from raw tile data (legs, Greeks, scores). Claude analysis adds strategy rationale, technical insights, theta decay curves, and risk scenarios — but only to Firestore. The PDF and video script never see this analysis. Investors reading the PDF get a different (weaker) explanation than what the app shows.

2. The Principle: Analysis First

Claude analysis must happen before any output is generated. All 5 outputs read from the same enriched data. One source of truth, zero inconsistency.

ANALYSIS-FIRST PIPELINE

Key insight: The enriched pick JSON is an intermediate artifact that combines raw tile data + Claude analysis. Every downstream output reads from this artifact. If Claude generates a brilliant insight about gamma wall positioning, it appears in the PDF, the video script, the pick card, AND the app — all identical.

3. The 5 Outputs

Every weekly publish produces exactly these 5 artifacts. All generated from the same enriched data.

📄
PDF Report
Per-pick deep analysis. Sent to subscribers, embedded in newsletters. Includes strategy rationale, payoff diagram, risk analysis.
🎥
Video Script
Narration script for creative team. References same analysis: "We chose iron condor because gamma walls at 215/245 create a natural range..."
📊
Pick JSON
Machine-readable pick data for tracking. Powers the /picks pages, performance tracking, win rate calculations.
🔥
Firestore Tiles
weeklyPicks/{weekId} document. Investors build portfolios from these. Includes legs, Greeks, P&L metrics.
🤖
Firestore Analyses
analyses/{tileId} documents. Powers 3 tabs: "Why This Trade?", "Technical Analysis", "Theta & Risk". The richest content.

4. Pipeline Stages — In Order

The weekly publish command: npm run weekly. Runs all stages sequentially. Each stage depends on the previous one.

1
Stage 1 — Select & Bundle
create-weekly-picks.js
Read active tiles from Firestore. Bundle into weeklyPicks/{weekId}. Mark previous weeks as "past".

Input: Firestore tiles (isActive: true)
Output: Firestore weeklyPicks/{weekId} with tileIds, picks summary, theme, dateRange
What's NOT here yet: No analysis, no PDF, no video. Just the raw pick selection.
2
Stage 2 — Claude Deep Analysis
analyse-tiles.cjs
For each pick, call Claude CLI with full tile context (legs, Greeks, gamma walls, spot price, DTE). Claude generates structured JSON with 4 analysis sections. Save to local JSON + Firestore.

Input: Firestore tiles/{tileId} (the picks from Stage 1)
Output:
  • Local: pipeline/output/{weekId}/{tileId}.json (enriched pick = tile + analysis merged)
  • Firestore: analyses/{tileId} (for app's 3 detail tabs)

The enriched pick JSON contains:
  • All tile data (symbol, strategy, legs, Greeks, gamma walls, P&L metrics)
  • strategyRationale — why this strategy, strikes, expiry + alternatives
  • technicalIndicators — RSI, Bollinger, MACD, MAs, IV, support/resistance
  • thetaDecaySchedule — daily theta curve, early close recommendation
  • riskAnalysis — max pain, earnings risk, management plan

This is the single source of truth for all downstream outputs.
3
Stage 3 — Generate Outputs
generate-outputs.js (new unified script)
Read the enriched pick JSONs from Stage 2. Generate all 3 remaining outputs:

3a. PDF Reports (per pick)
Full analysis baked in: strategy rationale, payoff diagram, theta decay chart, risk scenarios, management plan.
Output: pipeline/output/{weekId}/pdf/{SYMBOL}-{strategy}.pdf

3b. Video Script (per week)
Narration script referencing Claude's analysis: "For Adobe, we're using an iron condor because the gamma walls at $215 and $245 create a defined range, and with IV rank at 42, premium selling is favored..."
Output: pipeline/output/{weekId}/video-script.md

3c. Pick JSON Archive (per week)
Machine-readable snapshot for performance tracking. Includes analysis summary for each pick.
Output: pipeline/output/{weekId}/picks.json
npm run weekly = Stage 1 → Stage 2 → Stage 3. One command, all 5 outputs, perfectly in sync.

7. Target npm Scripts

Clean, composable commands for the weekly workflow.

Command What it does Stages
npm run weekly Full weekly publish. Select picks → Claude analysis → generate all outputs. 1 → 2 → 3
npm run picks:create Stage 1 only. Bundle tiles into weeklyPicks. 1
npm run analyse Stage 2 only. Run Claude analysis on active tiles. 2
npm run analyse -- --id {tileId} Analyse a single tile (re-run if needed). 2 (single)
npm run outputs Stage 3 only. Generate PDFs + video script + pick JSON from enriched data. 3
npm run picks:close Friday close. Calculate P&L, mark week as past. Close
npm run picks:preview Dry run — show what would be selected without publishing. 1 (dry)
