Root Cause — Why Data Is Out of Sync

Problem 1: analyse-tiles.cjs doesn't update tiles collection

When analysis runs, sentiment is saved to analyses/{tileId} and enriched-pick.json but never pushed back to tiles/{tileId}. The React frontend reads from tile.sentiment, so tiles published before the 4-engine update still have old/missing data.

Fix: Add tile.update() after analysis push (1 line)

Problem 2: generate-outputs.js strips sentiment from picks.json

The enriched-pick.json has complete 4-engine sentiment. But generate-outputs.js builds the picks.json archive without the sentiment field. This archive feeds PicksPage, WeekViewerPage, and RecapPage.

Fix: Include sentiment in picks.json output object

Problem 3: PDF/video generators don't know about sentiment

build-enriched-report-data.py creates 70+ template variables but zero sentiment vars. generate-script.py has 10 slides but no sentiment slide. The enriched data is there — it's just not being extracted.

Fix: Add SENTIMENT_* template vars + new PDF section + new video slide
