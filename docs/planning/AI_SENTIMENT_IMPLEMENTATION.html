Current Pipeline Architecture

NewLeaf runs 6 pipelines across two domains: the Scanner (data collection every 15 min) and the Publishing (manual analysis → picks → PDFs → video scripts).

 SCANNER PIPELINES (automated, cron-driven)
═══════════════════════════════════════════════════════════════════════

Pipeline 1: Intraday Fast  ── every 15 min, 9am-4pm ET Mon-Fri
  scanner/pipeline-fast.js --watchlist
  ├─ Alpaca API → spot price, bars, option chains (bid/ask/IV/Greeks)
  ├─ Technical analysis → RSI, Bollinger Bands, SMA 50/100/200
  ├─ Gamma wall detection → put wall, call wall, band width
  ├─ Opportunity scoring → 3-pillar score (Gamma + IV + Trend = 0-100)

  ├─ Generate weekly + monthly watchlist snapshots

  ├─ Fetch latest.json from R2 for each watchlist symbol
  ├─ Transform v3 reports → tile format
  ├─ Deactivate old tiles, write new tiles/{tileId} to Firestore
  └─ Update marketState/current document

Where Sentiment Fits In

Sentiment plugs into the two publishing pipelines — publish-pick.cjs and analyse-tiles.cjs. It enriches the same data that powers tiles, analysis tabs, PDFs, and video scripts.

BEFORE (current):

  publish-pick.cjs:
    Alpaca data → build strategy → Claude analysis → Firestore

AFTER (with sentiment):

  publish-pick.cjs:
    Alpaca data → build strategy → fetchSentiment(symbol) → Claude analysis → Firestore

                                        │                          │  Key Drivers: MS upgrade...
                                        │                          │  Material Events: none

                                             score, label,          by real news context

                                             modifier, flags

  Result: PDFs, video scripts, and dashboard all inherit sentiment
  because they read from the same enriched tile + analysis data.
Downstream benefit: Because PDFs and video scripts are generated from the enriched-pick.json (which merges tile + analysis), adding sentiment to the tile and analysis prompt means PDFs automatically include sentiment context and video scripts reference real market news — zero changes to the PDF/script generators needed.

Scope: Published Tiles & Weekly Picks Only

Sentiment analysis runs only for symbols that are actually being used — published tiles in Firestore and weekly pick candidates. Not all 111 watchlist symbols.

Why not all 111? Most of the watchlist is background scanning. Only a handful of symbols (typically 5-15) get published as tiles or selected as weekly picks. Running sentiment for all 111 would cost $3-6/month for data nobody looks at. Scoped to published tiles + picks, we're looking at ~5-15 Claude calls/week = less than $1/month.

Architecture

Sentiment is a module, not a pipeline. It's called inline when tiles are published or picks are analysed.

TRIGGER 1: TILE PUBLISHING

  Workbench ─► publish-pick.cjs
                │
                ├─ Read tile data from Firestore
                ├─ NEW: fetchSentiment(symbol)  ──► Claude CLI + web search
                │     └─ returns { score, label, summary, drivers, events }
                ├─ Enrich tile with sentiment fields
                └─ Save to Firestore tiles/{tileId}


TRIGGER 2: WEEKLY PICKS ANALYSIS

  analyse-tiles.cjs ──► for each unanalysed tile:
                │
                ├─ Read tile from Firestore
                ├─ NEW: fetchSentiment(symbol)  ──► Claude CLI + web search
                │     └─ cached if fetched within 6 hours
                ├─ Enrich Claude analysis prompt with sentiment context
                ├─ Call Claude for deep analysis (existing)
                └─ Save analysis to Firestore analyses/{tileId}


CACHING

  reports/{SYMBOL}/sentiment.json  ──  6-hour TTL
    └─ If fresh cache exists, skip Claude call
    └─ If stale or missing, call Claude, save cache
    └─ Both triggers share the same cache
Key difference from previous plan: No separate pipeline-sentiment.js. Instead, a shared module (sentiment-engine.js) is called inline by publish-pick.cjs and analyse-tiles.cjs. The module handles caching so the same symbol is never fetched twice within 6 hours.

Claude Sentiment Prompt

One Claude CLI call per symbol. Uses web search to gather real-time news and analyst data.

PROMPT (per symbol)

You are a financial sentiment analyst for an options trading system.
Analyze the current market sentiment for {symbol} using web search.
Focus on developments from the last 48 hours.

Search for:
  1. Breaking news and developments
  2. Analyst upgrades, downgrades, price target changes
  3. Earnings announcements, guidance, pre-announcements
  4. Material corporate events (M&A, regulatory, legal)
  5. Social media and retail trader sentiment
  6. Sector-wide themes affecting this stock

Return ONLY a JSON object (no markdown, no explanation):
{
  "symbol": "{symbol}",
  "score": 72,              // 0-100. 50=neutral, >70=bullish, <30=bearish
  "label": "bullish",        // bullish | neutral | bearish
  "confidence": 0.85,        // 0.0-1.0 based on source quality
  "summary": "...",           // 2-3 sentence synthesis
  "keyDrivers": [
    {"factor": "Morgan Stanley PT raise to $260",
     "impact": "positive",     // positive | negative | neutral
     "source": "Bloomberg"}
  ],
  "materialEvents": [],      // earnings, M&A, regulatory → triggers suppress
  "socialSentiment": "...",   // brief note on retail/social mood if found
  "sectorContext": "..."      // broader sector theme affecting this stock
}
CLI invocation — same proven pattern as analyse-tiles.cjs line 221:

spawnSync('claude', ['--print', '--output-format', 'text'], { input: prompt, timeout: 180000 })

Since we're doing individual symbols (not sector batches), each call is more focused and produces higher-quality sentiment. With caching, the same symbol won't be fetched again within 6 hours.

Sentiment Modifier Logic

Applied when tiles are published. Adjusts the opportunity score by -5 to +5 points.

ConditionActionModifierFlag
Sentiment ≥ 75, aligned with trend Boost +1 to +5 Shown on tile card
Sentiment < 35, contradicts bullish trend Caution -2 caution — divergence warning
Sentiment < 30, confirms bearish trend Bearish -3 Displayed in analysis
Material event (earnings, M&A, regulatory) Suppress 0 suppress — skip from picks
Cache miss or low confidence (< 0.5) None 0 Graceful degradation
Suppress logic for picks: If materialEvents is non-empty (e.g. earnings in 5 days, pending M&A), the tile is flagged with suppress. In analyse-tiles.cjs, suppressed tiles are skipped from weekly picks — they shouldn't be recommended when a binary event is imminent.
