Live P&L Tracking System

Architecture & data flow for real-time options portfolio P&L calculation
NewLeaf System · Design Document · April 2026 · v1.0

1. Overview

The Live P&L system calculates unrealized profit/loss for portfolio positions in real time, entirely client-side. It uses a three-tier pricing model to estimate current option values from available data, compares them against entry premiums, and surfaces strategy health with actionable adjustment suggestions.

Key Principle

All P&L is calculated in-memory on the client using data from R2 (updated every 15 min by the pipeline). No Firestore writes are needed for P&L — the source of truth is entry data (stored once at portfolio add) + live market data (from R2).

4. Three-Tier Pricing Model

For each option leg, the system tries three methods in order of accuracy:

TierMethodSourceAccuracyWhen Used
Tier 1 R2 Option Chain Match R2 optionChain[].mid Best — actual market mid-price When R2 has the exact strike/expiry/type
Tier 2 Black-Scholes Estimate Current spot + entry IV Good — theoretical fair value When R2 match unavailable but IV is known
Tier 3 Intrinsic Value Current spot vs strike Approximate — ignores time value Fallback when no IV data available

P&L Calculation

// For each leg:
currentPremium = tier1(r2Chain) || tier2(blackScholes) || tier3(intrinsic)

// Leg P&L (per contract):
legPnl = (currentPremium - entryPremium) * multiplier * 100
  where multiplier = +1 for BUY, -1 for SELL

// Position P&L:
unrealizedPnl = sum(legPnl for all legs)

// Example: AAPL Iron Condor
// Sold 245 put: entry $3.10, current $1.20 → P&L = (1.20 - 3.10) * -1 * 100 = +$190
// Bought 240 put: entry $1.65, current $0.45 → P&L = (0.45 - 1.65) * +1 * 100 = -$120
// Total position P&L = sum of all 4 legs

Black-Scholes Implementation

// Client-side BS pricing (pnlCalculator.js)
function bsPrice(spot, strike, expiry, iv, type, r = 0.045) {
  const T = max(daysToExpiry / 365.25, 0.001)
  const d1 = (ln(S/K) + (r + iv²/2) * T) / (iv * √T)
  const d2 = d1 - iv * √T

  Call = S * N(d1) - K * e^(-rT) * N(d2)
  Put  = K * e^(-rT) * N(-d2) - S * N(-d1)
}

// Falls back to R2 IV if available and more recent

5. Strategy Status Engine

After calculating P&L, the system evaluates position health based on where the underlying sits relative to the short strikes, DTE, and P&L percentage. It returns a status label, color, and actionable suggestion.

Healthy
Price is within profit zone, away from short strikes. No action needed.
Take Profit
Captured >50% of max profit. Suggest closing early to free capital.
Theta Harvesting
≤7 DTE in profit zone. Theta is accelerating — hold or close near target.
Breached
Price has moved past a short strike. Suggest rolling the tested side.
Max Loss
Price beyond long strike — at maximum loss. Close to limit damage.
Expiring at Loss
DTE = 0 and in loss zone. Close immediately to avoid assignment.

Zone Detection Logic

// For Iron Condors / Spreads:
if (spot < shortPut.strike)
  → breached put side (or max loss if below longPut)
else if (spot > shortCall.strike)
  → breached call side (or max loss if above longCall)
else
  → profit zone

// Combined with DTE and P&L %:
if (zone === profit && pnlPct >= 50%)  → "Take Profit"
if (zone === profit && dte <= 7)       → "Theta Harvesting"
if (zone === breached && dte <= 5)     → "Critical — close or roll"
if (zone === max_loss)                 → "Close to limit damage"

8. Known Limitations

  • Client-side only — P&L is only calculated when a user has the app open. There is no background job updating P&L in Firestore.
  • IV staleness — Tier 2 uses IV from entry time. Actual IV may have changed significantly, especially around earnings or market events.
  • No real-time option prices — R2 updates every 15 min. Between updates, the P&L reflects the last scan's option prices.
  • Spread pricing — Black-Scholes prices individual legs independently. In practice, spreads trade as packages with different bid-ask dynamics.
  • Assignment risk — The status engine flags expiring positions but does not model early assignment probability for American options.

9. Future Enhancements

  1. Background P&L cron job — A Node script running alongside pipeline-fast.js that iterates all users' portfolios and updates P&L in Firestore. Enables push notifications and alerts when the app is closed.
  2. Live IV from R2 — Use the R2 report's current IV (from gammaData.ivData) instead of entry IV for more accurate BS estimates.
  3. Adjustment simulator — When a position is breached, show what rolling the tested side would look like (new strikes, new credit/debit, new risk profile).
  4. Historical P&L snapshots — Daily cron that snapshots portfolio P&L to Firestore for the Portfolio Growth chart.
  5. Broker integration — Fetch actual fill prices from Alpaca when trades are entered, replacing theoretical entry premiums.
