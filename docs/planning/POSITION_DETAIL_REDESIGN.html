Position Detail Page — Redesign

Fix stale data, add live P&L tracking, progress visualization, and actionable alerts

NewLeaf System · Design Spec · April 2026

1. Current Bugs

BUG: "Your Position" shows -$905 — R2 option chain matches wrong expiry (04-17 instead of 04-24). matchOptionLeg fixed to require exact expiry match, but PositionPnlCard fetches R2 independently and may not pass the correct expiry.
BUG: "Profit & Loss Profile" says "If AMD stays below $210" — wrong symbol (should be ADBE) and wrong strike (should be $220). Component uses hardcoded generic text instead of tile data.
BUG: Greeks (Theta: 0.00, Vega: 0.00) are from publish time. Never recalculated with current spot price and reduced DTE.
BUG: "Live Trade Example" premiums are entry premiums, not current. Label says "Based on current price" but shows stale data.
BUG: Risk Scenarios ("No Move: -$52") use publish-time spot, not current. Results are stale.
BUG: Strategy Legs show entry premiums only — no comparison with current values.

2. Redesigned Page Structure

The page should have a single data source: usePositionLiveData hook that fetches R2 data once and distributes current prices, recalculated Greeks, and live P&L to all sections.

Section Order (Overview Tab)

  1. Hero Header — symbol, live price, strategy badge, DTE countdown (existing, works fine)
  2. Position Monitor — NEW: replaces "Your Position" card. Progress bar, P&L, profit capture %, action alerts
  3. Strategy Summary — existing EnhancedStrategyHeader (works fine)
  4. Live Legs Table — entry vs current premium, per-leg P&L, live Greeks per leg
  5. P&L at Expiry — Highcharts payoff diagram (works fine)
  6. Risk Scenarios — recalculated with CURRENT price, not publish-time
  7. Exit Signals — calculated from actual strikes (fixed)

Data Sources for Legs Table

  • Entry premium — leg.entryPremium (stored at portfolio add time)
  • Current premium — Tier 1: matchOptionLeg(r2Chain, strike, expiry, type) with exact expiry. Tier 2: bsPrice() fallback.

  • Current Greeks — Recalculate with bsPrice() using current spot, current DTE, entry IV. Not the stale publish-time Greeks.

7. Implementation Plan

Step 1: Create usePositionLiveData hook

Single hook for the PositionDetail page that provides all live data:

const {
  currentSpot,         // from R2 or PriceContext
  r2Chain,             // option chain for leg matching
  pnlResult,           // from calculatePositionPnl
  strategyStatus,      // from getStrategyStatus
  liveGreeks,          // recalculated from BS
  liveLegDetails,      // entry vs current per leg
  riskScenarios,       // recalculated with current spot
  progressPct,         // position on max-loss ↔ max-profit scale
  profitCapturePct,    // % of max profit (or max loss) reached
} = usePositionLiveData(tile, portfolioItem);

Step 2: Build PositionMonitor component

Replaces PositionPnlCard. Contains progress bar, metric row, and action alert. Uses data from the hook.

Step 3: Build LiveLegsTable component

Replaces both LiveTradeExampleCard and the Strategy Legs section. Single table with entry vs current columns.

Step 4: Update Greeks section

Recalculate Greeks per leg using BS with current spot and remaining DTE. Show net position Greeks.

Step 5: Fix Risk Scenarios

Pass current spot instead of publish spot. Fix symbol reference (use tile.symbol not hardcoded).

Step 6: Fix P&L Profile

Replace hardcoded "AMD" with tile.symbol. Use actual short strikes for condition text.
