Pipeline Operating Guide

How to publish picks, generate outputs, and close weeks

1. Publish a Pick

The core command. Fetches live data from Alpaca, builds the strategy, runs Claude analysis, pushes to Firestore, and saves enriched JSON.

# Basic usage
node pipeline/publish-pick.cjs NVDA --strategy "iron condor" --expiry 2026-04-24

# Preview without publishing
node pipeline/publish-pick.cjs NVDA --strategy "iron condor" --expiry 2026-04-24 --dry-run

# Also generate PDF
node pipeline/publish-pick.cjs NVDA --strategy "iron condor" --expiry 2026-04-24 --pdf

What Happens (10 Steps)

  1. Fetch live spot price from Alpaca
  2. Fetch live option chain from Alpaca for the specified expiry
  3. Fetch gamma wall context from R2
  4. Auto-select strikes, build legs, calculate P&L, Greeks, breakevens, PoP
  5. Create fresh tile in Firestore tiles/{id}
  6. Run Claude CLI analysis (~60s) — 4 sections
  7. Push analysis to Firestore analyses/{id}
  8. Save enriched-pick.json locally
  9. Append to weeklyPicks/{weekId}
  10. Generate PDF (with --pdf)

Finding Valid Expiry Dates

If Alpaca returns 0 contracts, the date isn't a valid options expiry. Try the nearest Friday. Use --dry-run to verify first.

2. Weekly Workflow

Mon – Thu: Publish Picks (one at a time)

# Day 1
node pipeline/publish-pick.cjs NVDA --strategy "iron condor" --expiry 2026-04-24
node pipeline/publish-pick.cjs AAPL --strategy "iron butterfly" --expiry 2026-05-01

# Day 2
node pipeline/publish-pick.cjs SPY --strategy "iron condor" --expiry 2026-04-24
node pipeline/publish-pick.cjs QQQ --strategy "bull put spread" --expiry 2026-04-24

Each pick is immediately live in the React app (~60s per pick).

After All Picks: Generate Week Outputs

npm run outputs               # video script + pick JSON
node pipeline/generate-outputs.js --pdf  # + PDFs

Produces: picks.json, video-script.md, pdf/

Friday: Close the Week

npm run picks:close

Marks week as "past", deactivates tiles, calculates P&L.
