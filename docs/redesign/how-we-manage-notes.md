# /how-we-manage — Trade Lifecycle Visualisation

_Built 2026-04-19. Phase 2 of 4 visualisation pages._

## What was built

An interactive marketing page at `/how-we-manage` where users control a time-scrubber to step through a 45-day Iron Condor lifecycle. As they drag, the payoff chart, vitals, verdict pill, decision tree, and adjustment picker all update in real time.

### Architecture

```
src/marketing/how-we-manage/
├── HowWeManagePage.jsx    — page layout, all sub-components, state management
└── demoTradeData.js       — 46-session ADBE Iron Condor lifecycle data
```

Single state variable (`sessionIndex`) drives everything. No GSAP, no D3, no Canvas — all DOM/SVG via React state + Highcharts.

### Components in HowWeManagePage.jsx

| Component | Purpose |
|---|---|
| `TimeScrubber` | Draggable timeline with verdict colour bands, Play/Prev/Next, day counter |
| `VitalTile` | Spot price, delta, DTE, P&L — updates live on scrub |
| `DecisionTree` | 5-state rule hierarchy with active-path highlighting |
| `AdjustmentPreview` | Slides in during ACTION_NEEDED/EXIT, shows recommended adjustment |
| `VerdictPill` | Colour-coded state indicator (inline in vitals section) |
| `CaptionCard` | Narrator text — 12 captioned key moments, interpolated between |

### Trade data

**Synthetic ADBE Iron Condor** — $500/$510 puts, $540/$550 calls, entry at $525, 45 DTE.

Verdict arc (derived from the real `evaluate()` function, not hardcoded):

| Days | Verdict | Trigger |
|---|---|---|
| 0-15 | ON_TRACK | Delta below 0.25, theta collecting |
| 16-23 | MONITOR | Delta crosses 0.25 |
| 24-31 | ACTION_NEEDED | 21-DTE escalation (MONITOR → ACTION_NEEDED) |
| 32-36 | EXIT | Delta 0.42 at $544 gap + 21-DTE escalation |
| 37-42 | ACTION_NEEDED | Recovery but still under 21-DTE pressure |
| 43-45 | TAKE_PROFIT | 51% of max profit captured |

All verdicts are generated by importing and calling `evaluate()` from `verdictEngine.js` — not duplicated.

### Interaction model

- **Drag scrubber**: updates sessionIndex → all components re-render
- **Play button**: auto-advances at 2 sessions/second (500ms interval)
- **Prev/Next buttons**: advance by one session
- **Keyboard**: Arrow keys advance/reverse, Space toggles play
- **Touch**: scrubber responds to touch drag events

### Key moments

| Day | What happens visually |
|---|---|
| 0 | Entry. Chart shows full iron condor. All vitals green. |
| 16 | MONITOR fires. Verdict pill turns amber. Decision tree highlights delta threshold. |
| 24 | ACTION_NEEDED fires. 21-DTE escalation. Tree highlights escalation rule. |
| 32 | EXIT fires. ADBE gaps to $544. Delta 0.42. Adjustment picker slides in from right. |
| 35 | Recovery begins. Delta drops to 0.30. Verdict de-escalates to ACTION_NEEDED. |
| 43 | TAKE_PROFIT fires. 51% captured. Verdict pill turns green. |

## Approximations

1. **Trade data is synthetic** — designed to hit specific transitions. Not a real historical trade. Disclosed in the page copy as an illustrative example. The data file (`demoTradeData.js`) is documented for easy swap to real data.

2. **Adjustment recommendations** — computed from the real `getValidAdjustments()` function but use estimated pricing (same approximations as the in-app catalogue). Real R2 chain pricing is not available on this marketing page.

3. **Payoff chart** — shows the position at current spot but does not account for time decay or IV changes in the payoff curve. The chart shows at-expiration payoff, not mark-to-market.

4. **Decision tree** — simplified to show the 5 verdict states and their key rules. The full engine has per-strategy rules, universal overrides, and session-breach counting that aren't all shown in the tree.

## Known issues

1. **Scrubber on mobile** — touch drag works but lacks haptic feedback and snap-to-session. The handle is small (16px) — may need enlarging on touch devices.

2. **Caption interpolation** — only 12 of 46 sessions have custom captions. Between captioned sessions, the most recent caption persists. Adding more captions for intermediate sessions would improve the narrative flow.

3. **Verdict transitions are instant** — no animated transition between verdict states (colour snaps). Adding a 200ms colour transition would improve the feel, but is suppressed by `prefers-reduced-motion` check.

4. **The Day 32 EXIT moment** — the adjustment picker slides in, which is the dramatic moment. But the exit signal could be more visually dramatic (flash, pulse, screen shake). Kept minimal for now.
